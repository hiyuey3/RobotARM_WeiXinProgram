# 舵机控制小程序

这是一个基于微信原生框架的舵机控制小程序，使用 MQTT 协议通过 WebSocket 与设备通信。

## 功能特性
 - 通过 MQTT 协议控制三个舵机（舵机0, 1, 2）
 - 实时滑动条控制，滑动结束即发送指令
 - 订阅设备反馈主题，实时显示状态
 - 显示视频流（使用 image 标签）
 - 一键复位所有舵机到 90 度

## 技术参数

### MQTT 连接配置

- **连接地址**: `ws://mqtt.5i03.cn:8083/mqtt`
- **ClientID 格式**: `wx_client_xxxx`（自动随机生成）
- **下行主题**: `/iot/device/{ESP32_MAC地址}/down`
- **上行主题**: `/iot/device/{ESP32_MAC地址}/up`

### 控制指令格式

发送到下行主题的 JSON 格式：

```json
{
  "servo": 0,    // 舵机编号: 0, 1, 或 2
  "angle": 90    // 角度值: 0-180
}
```

### 反馈数据

从上行主题接收的数据将直接显示在状态栏中。

## 使用步骤

### 1. 开发环境配置

在微信开发者工具中，需要进行以下设置：

1. 打开项目设置
2. 找到「本地设置」
3. **勾选「不校验合法域名、web-view（业务域名）、TLS 版本以及 HTTPS 证书」**

> **重要**: 开发阶段必须勾选此选项，否则无法连接到 MQTT 服务器

### 2. 真机调试配置

如果需要在真机上调试，需要在小程序管理后台配置合法域名：

1. 登录 [微信公众平台](https://mp.weixin.qq.com/)
2. 进入「开发」-「开发管理」-「开发设置」
3. 找到「服务器域名」配置
4. 在「socket合法域名」中添加: `wss://mqtt.5i03.cn`

> 注意：由于当前使用的是 `ws://` 协议（非 wss），真机调试可能需要将服务器升级为 wss 协议

### 3. 配置设备

在小程序中输入以下信息：

1. **设备MAC地址**（即设备号）: 输入你的设备 MAC 地址（例如：`AA:BB:CC:DD:EE:FF` 或 `AABBCCDDEEFF`）
   - 此 MAC 地址用于构建 MQTT 主题：`/iot/device/{MAC地址}/down` 和 `/iot/device/{MAC地址}/up`
2. **局域网 IP**: 输入设备的局域网 IP 地址（例如：`192.168.1.100`）
   - 此 IP 用于访问视频流：`http://{IP}:81/stream`
3. 点击「订阅设备反馈」按钮

### 4. 控制舵机

- 滑动「舵机0」、「舵机1」、「舵机2」的滑块来控制角度（0-180度）
- 滑动结束后会自动发送控制指令
- 点击「复位」按钮可将所有舵机恢复到 90 度

### 5. 查看状态

状态栏会显示：
- MQTT 连接状态
- 发送的控制指令
- 设备返回的反馈数据

## 核心代码说明

### MQTT 初始化

```javascript
// 在 onLoad 中自动连接
onLoad() {
  this.connectMQTT()
}
```

### 发送控制指令

```javascript
// 滑动结束时触发
onServo0Change(e) {
  const angle = e.detail.value
  this.sendServoCommand(0, angle)  // 发送 {"servo": 0, "angle": angle}
}
```

### 接收反馈

```javascript
mqtt.onMessage((topic, message) => {
  if (topic === that.data.topicUp) {
    // 显示在状态栏
    that.setData({ statusText: `收到反馈: ${message}` })
  }
})
```

## 常见问题

### 1. 无法连接 MQTT 服务器

- 检查是否勾选了「不校验合法域名」
- 检查网络连接是否正常
- 确认 MQTT 服务器地址是否正确

### 2. 视频流不显示

- 确认局域网 IP 地址是否正确
- 确认设备的视频流服务是否启动（端口 81）
- 检查手机和设备是否在同一局域网

### 3. 控制指令无响应

- 确认已点击「订阅设备反馈」按钮
- 检查 MAC 地址是否正确
- 查看状态栏的反馈信息

### 4. 真机调试问题

- 真机调试需要配置合法域名
- 建议使用 wss:// 协议而非 ws://
- 视频流的 http:// 地址在真机上可能无法访问，需要使用 https:// 或者通过业务域名配置
